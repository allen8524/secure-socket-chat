# Demo

## 목적

`demo.py`는 포트폴리오 시연을 위한 CLI 자동 데모입니다. GUI 창을 여러 개 띄우지 않아도 서버, 두 개의 테스트 클라이언트, 암호화 채널, TOFU 검증, sequence 증가, 파일 전송, SHA-256 무결성 검증, 서버 통계 흐름을 한 번에 확인할 수 있습니다.

이 데모는 기존 `run_server.py`와 `run_client.py` 실행 방식을 대체하지 않습니다. GUI 수동 시연은 실제 사용자 경험을 보여주기 위한 흐름이고, CLI 자동 시연은 평가자가 핵심 네트워크 흐름을 빠르게 확인하기 위한 보조 도구입니다.

## 실행

```bash
python demo.py
```

시나리오만 미리 확인하려면 socket을 열지 않는 dry-run 모드를 사용할 수 있습니다.

```bash
python demo.py --dry-run
```

## 자동 시나리오

1. 로컬 서버를 thread로 시작합니다.
2. `alice`, `bob` 테스트 클라이언트를 연결합니다.
3. TOFU 서버 fingerprint 검증 결과를 출력합니다.
4. `alice`가 전체 메시지를 보내고 `bob` 수신을 확인합니다.
5. `alice`가 `bob`에게 귓속말을 보내고 수신을 확인합니다.
6. 작은 샘플 텍스트 파일을 전송하고 SHA-256 무결성 `OK`를 확인합니다.
7. 서버 통계를 요청하고 online/messages/files 값을 출력합니다.
8. 모든 클라이언트 연결과 서버 socket을 정리합니다.

## 주의사항

- 데모는 임시 디렉터리에 TOFU trust store와 샘플 파일을 만들고 종료 후 정리합니다.
- 현재 서버 키는 연결 단위로 생성되므로, 데모 안에서 두 번째 클라이언트가 같은 host:port에 대해 fingerprint 변경 경고를 감지할 수 있습니다. CLI 데모는 이 상황을 출력하고 계속 진행합니다.
- private key, 전체 암호문, 전체 파일 binary는 출력하지 않고 요약 정보만 표시합니다.
- `127.0.0.1:9999` 포트가 이미 사용 중이면 명확한 실패 메시지를 출력합니다.
