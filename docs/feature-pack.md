# Feature Pack: Security Demo Layer

이 문서는 포트폴리오 시연에서 강조하기 위한 추가 기능을 정리합니다.

## 1. 공개키 fingerprint와 session id

클라이언트와 서버는 키 교환 과정에서 서로의 공개키를 확인합니다. 각 공개키는 SHA-256 기반 짧은 fingerprint로 변환되어 GUI에 표시됩니다.

표시 항목:

- cipher: `PyNaCl Box (Curve25519 XSalsa20-Poly1305)`
- session id
- client public-key fingerprint
- server public-key fingerprint

이 기능은 암호화가 내부에서만 동작하는 것이 아니라, 사용자가 현재 세션의 보안 상태를 확인할 수 있게 만들기 위한 기능입니다.

## 2. 이미지/파일 SHA-256 무결성 검증

이미지와 일반 파일 전송 시 송신자는 payload의 SHA-256 해시를 계산해 header에 포함합니다. 수신자는 payload를 받은 뒤 다시 SHA-256을 계산하고 header의 해시와 비교합니다.

결과는 채팅창에 다음처럼 출력됩니다.

```txt
SHA-256 무결성: OK / abcd1234...
```

## 3. 서버 통계 명령어

클라이언트는 `/stats` 명령어 또는 `서버통계` 버튼으로 서버 상태를 조회할 수 있습니다.

조회 항목:

- 서버 uptime
- 현재 접속자 수
- 누적 메시지 수
- 누적 이미지/파일 수
- 누적 이미지/파일 바이트 수
- 현재 접속자 목록

## 4. 대화 로그 저장

`/save` 명령어 또는 `대화저장` 버튼을 통해 현재 대화 로그를 텍스트 파일로 저장할 수 있습니다. 저장 파일 상단에는 현재 보안 세션 정보가 함께 기록됩니다.

## 면접 설명 포인트

이 프로젝트는 단순히 메시지를 주고받는 채팅 프로그램이 아니라, 네트워크 프로토콜과 보안 상태를 사용자가 확인할 수 있도록 만든 시연형 보안 채팅 프로그램입니다. 자체 패킷 framing, 공개키 기반 암호화 채널, fingerprint 기반 세션 식별, SHA-256 파일 무결성 검증, 서버 상태 조회를 직접 구현했습니다.
